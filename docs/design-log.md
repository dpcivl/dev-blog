# 블로그 디자인 결정 로그

스킨 / 디자인 관련 모든 결정을 시간순으로 기록. 미래에 다시 스킨을 바꿀 때 **"왜 이렇게 했는지"** 를 한곳에서 확인할 수 있도록.

## 사용법

- 새 결정이 생길 때마다 **맨 위 "결정 로그"** 에 새 항목 추가 (역시간순)
- 취소된 결정은 지우지 말고 **취소선 + 취소 이유** 명시
- **Current Tokens** (맨 아래) 는 항상 현재 적용된 값으로 유지

## 결정 로그

### 2026-06-17 — 첫 모던 리디자인 시작

와이프의 "옛날 블로그 같다" 피드백을 계기로 본격 리뉴얼 시작. [redesign-plan.md](./blog-redesign-plan.md) 의 Phase 단위 절차에 따라 진행.

#### 결정 1 — 탭 구성: 5개

**선택**: `포스트` · `시리즈` · `플레이그라운드` · `태그` · `검색` (+ 테마 토글 아이콘)

**이유**:
- About 페이지는 새 사이드바에 프로필+바이오가 들어가므로 nav 에서 제외 (중복 회피)
- 시리즈(UI/DB 어휘 정리)와 플레이그라운드(인터랙티브 데모)는 이 블로그의 차별 자산
- 검색은 아이콘으로 가볍게

**대안**: About 유지 / Archives 추가 — 둘 다 제외 (사이드바·태그가 그 역할 흡수)

#### 결정 2 — 폰트: Pretendard

**선택**: 본문/헤딩 모두 **Pretendard** (300, 400, 500, 600, 700, 800 weight). 코드는 기존 Google Sans Code 유지.

**이유**:
- 한·영 모두 자연스러움 (한국 모던 블로그 표준)
- 가변 폰트, 무료, 안정적 CDN(jsdelivr) 제공
- 작성자가 와이프 피드백 중 "딱딱함" 을 지적 → Pretendard 의 부드러운 곡선이 적합

**대안**:
- Spoqa Han Sans Neo (살짝 더 클래식)
- Wanted Sans (가장 최신, 그러나 안정성 측면 아직 검증 중)

#### 결정 3 — 컬러: 쿼츠 듀얼 시스템 (A안 채택)

작성자가 `#d8d4ea` (quartz) 를 액센트로 원함. 그러나 단일 사용 시 라이트 모드에서 가독성 실패(흰 배경 위 명도 대비 1.3:1, WCAG 미달).

**선택**: **라이트 모드는 듀얼**, **다크 모드는 쿼츠 그대로**.

| 모드 | 강(텍스트/링크) | 배경(태그·hover) |
|---|---|---|
| 라이트 | `#6a5acd` (slateblue) | `#d8d4ea` (quartz) |
| 다크 | `#d8d4ea` (quartz) | — |

**이유**:
- 라이트: 흰 배경 위에선 쿼츠가 안 보임. 같은 라벤더 계열의 진한 슬레이트블루로 보강
- 다크: 어두운 배경 위에선 쿼츠가 13:1 이상 대비 → 작성자 선호 그대로 사용 가능
- **톤(라벤더 계열)은 두 모드에서 동일**하게 유지

**대안**:
- ~~B안: `#d8d4ea` 단일~~ — 라이트 모드 접근성 실패로 기각
- ~~C안: 진한 보라(#7c3aed) 단일~~ — 쿼츠 느낌 손실

#### 결정 4 — 배경 컬러

| 모드 | 본문 배경 | 표면(카드/사이드바) |
|---|---|---|
| 라이트 | `#fafafa` (살짝 따뜻한 화이트) | `#ffffff` |
| 다크 | `#0f1115` (짙은 네이비-블랙) | `#1a1d23` |

**이유**: 순흑 `#000000` 보다 살짝 푸른빛 도는 다크가 눈에 편함. 라이트도 순백 `#ffffff` 본문 배경은 답답함 줄 수 있어 살짝 톤다운.

#### 결정 5 — 레이아웃: 사이드바 + 메인 2컬럼

작성자가 그려준 와이어프레임 기준:

```
┌──────────────────────────────────────┐
│  로고      [포스트][시리즈][플레이][태그][🔍][🌙] │
├────────┬─────────────────────────────┤
│ 프로필  │                              │
│ 사진    │                              │
│  ○      │      포스트 영역              │
│        │                              │
│ 자기소개 │                              │
│        │                              │
│ SNS    │                              │
└────────┴─────────────────────────────┘
```

- 좌측 사이드바: 좁게 (~240px 예정)
- 우측 메인: 가변
- 모바일: 사이드바가 위로 접힘 (예정)
- 전체 최대 폭: ~1200px

**이유**: 작성자가 그린 그림에 맞춰 + 모던 블로그에서 흔히 보이는 정적 사이드바 형태.

#### 결정 6 — 포스트 본문: 맨위/맨아래 이동 버튼

긴 글(NDT 글 같은) 에서 네비 편의성. 우측 하단 fixed floating button.

#### 결정 8 — 다크 모드 액센트 보정 (Phase 1 follow-up)

Phase 1 배포 후 작성자 확인: **다크 모드에서 본문(`#e8e8e8`)과 액센트(`#d8d4ea` 쿼츠)가 거의 같은 명도(91% vs 88%)라 제목·링크가 본문과 구분이 안 됨.**

**변경**: 다크 모드 `--accent` 만 `#d8d4ea` → **`#bb9af7`** (Tokyo Night 라벤더) 로 교체.
- 명도 약 79% → 본문과 12% 차이 → 위계 분명
- 작성자가 좋아하는 부드러운 라벤더 톤은 유지
- 검증된 개발자 다크 테마 색감 (Tokyo Night)

**고려한 대안**:
- ~~`#a78bfa` (Tailwind violet-400)~~ — 더 진하지만 약간 비비드해서 보류
- ~~`#cba6f7` (Catppuccin Mauve)~~ — 가장 안전하지만 다시 본문과 너무 가까워질 위험

**라이트 모드는 그대로** (`#6a5acd` 슬레이트블루는 본문 `#1a1a1a` 와 충분히 대비됨).

**교훈**: WCAG 명도 대비(텍스트 vs 배경)는 통과해도, **같은 영역 안에서 텍스트끼리의 위계** 는 별도 검증 필요. 다음에 색을 바꿀 때는 텍스트끼리의 명도 차이도 점검할 것.

#### 결정 7 — 작업 순서

원래 Phase 5단계 계획에서 작성자 와이어프레임이 큰 구조 변경을 요구해서 단계 재조정:

```
Phase 1  — 컬러 토큰 + Pretendard 폰트 (오늘)
Phase 2  — 헤더 재구성 (로고 좌 + 탭 우)
Phase 3  — 사이드바 + 메인 2컬럼 (큰 작업)
Phase 4  — 포스트 카드 디자인 (간격 여유)
Phase 5  — 글 본문 + 맨위/맨아래 버튼
Phase 6  — 시리즈/플레이그라운드 인덱스 페이지 신규
Phase 7  — 회귀 테스트 + 마무리
```

각 Phase 끝나면 이 로그에 결과·이슈 추가.
